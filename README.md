# echoed.uk

> 在噪音中，寻找信号。一个安静的个人策展站。

一个静态个人网站，把读到、听到、想过之后值得留下的东西，作为「回响」分享出来。
核心价值：**共鸣 > 噪音 · 深度 > 广度 · 持久 > 短暂**。

## 主要页面

| 文件 | 页面 | 说明 |
| --- | --- | --- |
| `index.html` | 首页 | 回声 Hero（canvas 波纹，可点击）+ 宣言 + 精选回响 + 订阅页脚 |
| `explore.html` | 探索回响 | 可按类型（文章 / 观点 / 对话 / 书）筛选的策展精选时间线 |
| `about.html` | 关于 | 品牌故事 |
| `echo-*.html` | 回响详情 | 每篇文章独立成页（含阅读进度条），见下方内容清单 |

纯静态，无构建步骤、无依赖、无框架。每个页面自包含（内联 CSS + 原生 JS），可单独用浏览器打开。字体经 Google Fonts 引入：思源宋体 Noto Serif SC（中文）+ Fraunces（拉丁字标）。

## 功能特性

- **日间 / 夜间主题切换**：导航栏右侧按钮切换，偏好存入 `localStorage`，并同步 `theme-color`；首屏内联脚本提前应用主题，避免闪烁。
- **PWA / iOS 主屏支持**：`site.webmanifest` + apple-touch-icon + 各机型启动画面（`assets/splash-*.png`），消除冷启动白屏；独立模式下导航栏显示刷新按钮。
- **阅读体验**：详情页顶部阅读进度条；正文中「原文/来源」用安静的灰色，「我的点评（回响）」用带暖色左缘的样式强调——这是品牌隐喻。
- **可发现性**：`rss.xml`（订阅源）与 `sitemap.xml`（站点地图）。
- **无障碍与适配**：响应式到手机、键盘焦点、`prefers-reduced-motion`。

## 部署

任意静态托管都可以。当前通过 **GitHub Pages 从 `master` 分支根目录** 发布——推送到 `master` 后会自动重新构建，约 1～2 分钟后生效。

- **GitHub Pages**：仓库 Settings → Pages → Source 选 `master` 分支根目录 → 保存。
- **Cloudflare Pages**：连接此仓库，构建命令留空，输出目录填 `/`。

绑定自定义域名 `echoed.uk` 后，把 DNS 指向托管商即可。`push.sh` 为一键推送脚本。

## 协作约定（与 Claude 的工作规则）

站主与 Claude 的固定约定，每次开工先读这一节，并据此执行：

1. **沟通语言**：用中文交流。
2. **分支与部署**：
   - 所有改动先在功能分支 `claude/website-review-o0eglu` 上开发、提交、推送。
   - 再 `git checkout master && git merge --ff-only` 合并到 `master`，推送 `master`。
   - GitHub Pages 从 `master` 根目录发布，约 1～2 分钟生效。**未经许可不推到其它分支。**
   - 推送统一 `git push -u origin <branch>`；网络失败按 2s/4s/8s/16s 退避重试，最多 4 次。
   - **不主动建 Pull Request**，除非站主明确要求。
3. **新增 / 改动文章的同步清单**（缺一不可）：
   - 新文件以 `echo-<slug>.html` 命名，**直接复制最新一篇文章做模板**，保证 head、PWA 启动图、导航、页脚、阅读进度条、主题切换、分享按钮完全一致。
   - 改 `<title>`、`description`（meta / og / twitter / 首屏导语四处一致）、`canonical`、`og:url`、`article:published_time`、og/twitter 标题、顶部 meta 行（类型 · 子类 · 日期 · 阅读时长）。
   - 同步更新 `explore.html`（时间线卡片置顶）、`sitemap.xml`、`rss.xml`（item 置顶 + `lastBuildDate`）、本 `README.md` 内容清单与篇数。
   - 详情页底部「下一条回响」做交叉互链。
4. **每篇文章都带分享按钮**：微信 / 朋友圈（优先 `navigator.share()` 系统原生分享——手机上一键唤起微信，可发给朋友或分享到朋友圈；不支持时回退到复制链接 + 提示）、微博（`service.weibo.com/share`）、复制链接；位置在文末署名（`.endnote`）之后、「下一条回响」（`.next`）之前。
   - 技术限制：静态网页**无法**让外部浏览器直接打开微信并自动发朋友圈（腾讯不开放此能力）。Web Share API 是手机端最接近「点一下跳微信直接分享」的方案；要做到打开即带分享卡片、自动填充，必须在微信内打开页面并接入公众号 JS-SDK（需服务端签名），属于另一套工程，暂未做。
5. **设计与品牌隐喻**：严格遵循上文「设计约定」——信号暖琥珀是唯一强调色、克制使用；「原文/来源」用安静灰、「我的点评/回响」用暖色左缘强调；保持响应式、键盘焦点、`prefers-reduced-motion`。
6. **个人记述要前后一致**：凡涉及站主本人或家人的私人事实，以下方「作者事实档」为准，不得与既有文章冲突。
7. **每一次变动都写进 README**：每次改动后，更新下方「变更记录」，并同步本节规则（本条即由站主于 06·26 提出）。
